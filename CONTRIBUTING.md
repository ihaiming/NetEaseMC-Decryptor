# 贡献指南

感谢你对本项目的兴趣！在提交贡献之前，请花几分钟阅读以下指南

## 提交 Issue
- 使用 Issue 模板（如果有）
- 清晰描述问题或建议
- 如果是 Bug，请提供复现步骤和环境信息

## 提交 Pull Request

### 基本原则
- **重大改动先开 issue 讨论**：涉及架构调整、技术栈变更、UI 重设计等，请先开 issue 讨论，达成共识后再动手
- **保持项目定位**：本项目坚持**轻量、原生、零依赖**，任何引入外部依赖的改动都需要充分理由
- **尊重原设计**：UI 和交互风格是项目的一部分，大幅改动前请先沟通

### PR 流程
1. fork 本仓库
2. 创建功能分支 (`git checkout -b feature/your-feature`)
3. 提交改动 (`git commit -m 'add: 某个功能'`)
4. 推送到你的 fork (`git push origin feature/your-feature`)
5. 打开 Pull Request，目标分支为 `main`

### PR 描述要求
- 说明改动的目的和内容
- 如果是修复 Bug，请关联相关 issue
- 如果是新功能，请说明使用场景

## 代码风格
- 保持原生 JavaScript，不引入不必要的依赖
- 使用清晰的变量命名，适当添加注释
- 保持单文件结构，除非有充分理由拆分

## 沟通礼仪
- 请保持友善、尊重的沟通态度
- 对代码有异议时，请聚焦代码本身，不要评价作者
- 欢迎任何建设性的反馈

感谢你的理解和支持！
